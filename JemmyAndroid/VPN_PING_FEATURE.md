# Функция пинга VPN серверов

## Что добавлено

Реализована полноценная функция проверки доступности VPN серверов (пинг).

### Новые компоненты:

1. **VpnPinger.kt** - Утилита для пинга серверов
   - Проверка доступности хоста и порта
   - Измерение задержки (latency)
   - Поддержка множественных пингов
   - Таймаут 5 секунд

2. **Обновленная модель VpnConfig**
   - `lastPingTime` - время последнего пинга
   - `lastPingLatency` - задержка в миллисекундах
   - `isReachable` - доступен ли сервер

3. **Обновленный UI**
   - Кнопка пинга для каждого конфига
   - Отображение результатов пинга
   - Индикатор загрузки при пинге
   - Автоматический пинг при добавлении конфига

## Как использовать

### В UI приложения:

1. **Добавление конфига:**
   - При добавлении нового VPN конфига автоматически выполняется пинг
   - Результат сохраняется и отображается

2. **Ручной пинг:**
   - Нажмите кнопку обновления (🔄) рядом с конфигом
   - Дождитесь результата (индикатор загрузки)
   - Результат отобразится под названием конфига

3. **Отображение результатов:**
   - ✓ 45ms • 10с назад - сервер доступен, задержка 45мс
   - ✗ Недоступен • 30с назад - сервер недоступен

### Программно:

```kotlin
// Простой пинг
val result = VpnPinger.pingVlessServer(vlessUrl)
if (result.isReachable) {
    println("Latency: ${result.latency}ms")
} else {
    println("Error: ${result.error}")
}

// Множественный пинг (3 раза)
val results = VpnPinger.pingMultiple(vlessUrl, count = 3)
val avgLatency = VpnPinger.getAverageLatency(results)
println("Average latency: ${avgLatency}ms")
```

## Технические детали

### Как работает пинг:

1. Парсит VLESS URL для получения хоста и порта
2. Пытается установить TCP соединение
3. Измеряет время подключения
4. Возвращает результат с задержкой или ошибкой

### Параметры:

- **Таймаут:** 5000ms (5 секунд)
- **Порт по умолчанию:** 443 (если не указан в URL)
- **Протокол:** TCP Socket

### Что проверяется:

- ✅ Доступность хоста
- ✅ Открытость порта
- ✅ Время отклика
- ❌ НЕ проверяется валидность VLESS конфигурации
- ❌ НЕ проверяется аутентификация

## Ограничения

1. **Только TCP проверка:**
   - Проверяется только доступность порта
   - Не проверяется работоспособность VLESS протокола

2. **Не гарантирует работу VPN:**
   - Сервер может быть доступен, но конфиг неверный
   - UUID может быть неправильным
   - Путь WebSocket может не работать

3. **Зависит от сети:**
   - Требуется интернет соединение
   - Может не работать в некоторых сетях
   - Firewall может блокировать проверку

## Пример использования

```kotlin
// В ViewModel или Repository
class VpnViewModel {
    suspend fun checkServerStatus(config: VpnConfig): PingResult {
        return VpnPinger.pingVlessServer(config.vlessUrl)
    }
    
    suspend fun checkAllServers(configs: List<VpnConfig>) {
        configs.forEach { config ->
            val result = VpnPinger.pingVlessServer(config.vlessUrl)
            configManager.updatePingResult(
                config.id,
                result.latency,
                result.isReachable
            )
        }
    }
}
```

## Troubleshooting

**Пинг всегда неудачный:**
- Проверьте интернет соединение
- Убедитесь что URL правильный
- Проверьте что порт открыт
- Попробуйте другую сеть

**Долгий пинг:**
- Нормально до 5 секунд (таймаут)
- Может быть медленная сеть
- Сервер может быть далеко

**Пинг успешный, но VPN не работает:**
- Это нормально - пинг проверяет только доступность
- Проверьте правильность UUID
- Проверьте параметры шифрования
- Проверьте путь WebSocket
