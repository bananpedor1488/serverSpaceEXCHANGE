# 🧪 Тестирование функций приватности

## Подготовка

### 1. Запуск сервера
```bash
cd JEMMY-SERVER
npm install
npm run start:dev
```

### 2. Проверка схем MongoDB
Убедитесь, что MongoDB запущена и схемы применены:
- `Identity` с полем `privacy_settings`
- Новая коллекция `BlockedUser`

### 3. Запуск iOS приложения
Откройте проект в Xcode и запустите на симуляторе или устройстве.

## 🧪 Тесты локальной безопасности

### PIN-код
1. ✅ Открыть: Профиль → Приватность → PIN-код
2. ✅ Установить 4-значный PIN (например: 1234)
3. ✅ Подтвердить PIN
4. ✅ Закрыть приложение
5. ✅ Открыть приложение → должен запросить PIN
6. ✅ Ввести неверный PIN → ошибка
7. ✅ Ввести правильный PIN → разблокировка
8. ✅ Удалить PIN-код

### Биометрия (Face ID / Touch ID)
1. ✅ Открыть: Профиль → Приватность
2. ✅ Включить Face ID / Touch ID
3. ✅ Закрыть приложение
4. ✅ Открыть приложение → запрос биометрии
5. ✅ Подтвердить → разблокировка

### Автоблокировка
1. ✅ Установить PIN-код
2. ✅ Открыть: Профиль → Приватность → Автоблокировка
3. ✅ Выбрать 1 минуту
4. ✅ Свернуть приложение на 1+ минуту
5. ✅ Открыть → должен запросить PIN

### Скрытие уведомлений
1. ✅ Включить: Профиль → Приватность → Скрыть контент уведомлений
2. ✅ Получить сообщение
3. ✅ Проверить уведомление → должно показывать только "Новое сообщение"

### Защита от скриншотов
1. ✅ Включить: Профиль → Приватность → Защита от скриншотов
2. ✅ Открыть чат
3. ✅ Попытаться сделать скриншот
4. ✅ Должно показать предупреждение (если реализовано)

### Автоудаление сообщений

**Тест:**
1. ✅ Установить: "24 часа"
2. ✅ Отправить сообщение
3. ✅ Через 24 часа сообщение должно удалиться (требует cron job)

  -H "Content-Type: application/json" \

    "settings": {
      "auto_delete_messages": 24

  }'

## 🚫 Тесты блокировки

### Заблокировать пользователя

**Через UI:**
1. ✅ Открыть профиль пользователя
2. ✅ Нажать "Заблокировать"
3. ✅ Подтвердить
4. ✅ Пользователь добавлен в список заблокированных

**Через API:**
```bash
curl -X POST http://localhost:3000/api/identity/block \
  -H "Content-Type: application/json" \
  -d '{
    "blocker_identity_id": "USER1_ID",
    "blocked_identity_id": "USER2_ID"
  }'
# Должен вернуть: {"success": true, "message": "User blocked"}
```

### Проверка блокировки

**Тест:**
1. ✅ Заблокированный пользователь пытается написать
2. ✅ Сообщение не доставляется
3. ✅ Заблокированный не видит профиль

**API тест:**
```bash
curl http://localhost:3000/api/identity/can-message/BLOCKED_ID/BLOCKER_ID
# Должен вернуть: {"can_message": false}
```

### Список заблокированных

**Через UI:**
1. ✅ Открыть: Профиль → Приватность → Заблокированные
2. ✅ Список показывает всех заблокированных

**Через API:**
```bash
curl http://localhost:3000/api/identity/blocked-list/USER_ID
# Должен вернуть: {"blocked_users": [...]}
```

### Разблокировать пользователя

**Через UI:**
1. ✅ Открыть список заблокированных
2. ✅ Нажать "Разблокировать"
3. ✅ Подтвердить
4. ✅ Пользователь удален из списка

**Через API:**
```bash
curl -X POST http://localhost:3000/api/identity/unblock \
  -H "Content-Type: application/json" \
  -d '{
    "blocker_identity_id": "USER1_ID",
    "blocked_identity_id": "USER2_ID"
  }'
# Должен вернуть: {"success": true, "message": "User unblocked"}
```
